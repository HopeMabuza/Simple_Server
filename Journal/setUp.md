# Minimum Node + Express Setup

Everything you need to create one GET endpoint that returns JSON. Nothing more.

---

## Step 1 — Set up the project

Open your terminal and run:

```bash
mkdir my-server && cd my-server
npm init -y
npm install express
```

What each command does:

- `mkdir my-server` — creates a new folder for your project
- `cd my-server` — moves into that folder
- `npm init -y` — creates a `package.json` file automatically (no questions asked)
- `npm install express` — installs Express into your project

---

## Already have a package.json? Skip Step 1

If you're working from an existing project that already has a `package.json`, you don't need `npm init -y` or `npm install express`. Just run:

```bash
npm install
```

That single command installs everything already listed in your `package.json` in one shot.

For example, if your `package.json` looks like this:

```json
{
  "name": "Server_Contract",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "dependencies": {
    "@account-abstraction/contracts": "^0.6.0",
    "@openzeppelin/contracts": "^4.2.0",
    "@openzeppelin/contracts-upgradeable": "^4.9.6",
    "cors": "^2.8.6",
    "dotenv": "^16.6.1",
    "ethereum-block-by-date": "^1.5.0",
    "ethers": "^6.16.0",
    "express": "^5.2.1",
    "hardhat": "^2.19.4"
  },
  "devDependencies": {
    "@nomicfoundation/hardhat-toolbox": "^4.0.0",
    "@openzeppelin/hardhat-upgrades": "^3.9.1"
  }
}
```

Here is what each relevant package does for your Express server:

- `express` — your server, handles all routes and endpoints
- `ethers` — talks to the blockchain from your server
- `dotenv` — loads secret keys from a `.env` file so they never hit the browser
- `cors` — allows your frontend (running on a different port) to call your server

The rest (`hardhat`, `@openzeppelin`, `@account-abstraction`) are for writing and deploying contracts — not needed for the server itself but fine to have in the same project.

Your setup flow becomes:

```bash
mkdir my-server && cd my-server
# paste your package.json into the folder, then:
npm install
# create index.js and start building
```

---

## Step 2 — Create your server file

Create a file called `index.js` in your project folder and paste this in:

```js
const express = require('express')
const app = express()

app.get('/api/hello', (req, res) => {
  res.json({ message: 'Hello from your server' })
})

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000')
})
```

What each line does:

- `require('express')` — loads Express into your file
- `app.get(...)` — creates a GET endpoint at the URL `/api/hello`
- `res.json(...)` — sends back a JSON response when that URL is called
- `app.listen(3000, ...)` — starts the server on port 3000

---

## Step 3 — Run the server

In your terminal:

```bash
node index.js
```

You should see:

```
Server running on http://localhost:3000
```

---

## Step 4 — Test it

Open your browser and go to:

```
http://localhost:3000/api/hello
```

Or test it from a second terminal window:

```bash
curl http://localhost:3000/api/hello
```

Either way, you'll get back:

```json
{ "message": "Hello from your server" }
```

## What you end up with

Three things in your project folder — two auto-generated, one you wrote:

```
my-server/
├── node_modules/    ← auto-generated by npm install
├── package.json     ← auto-generated by npm init
└── index.js         ← the file you wrote
```

---

## The pattern to remember

Every route you ever add to your dApp backend follows this same structure:

```js
app.get('/route', (req, res) => {
  res.json({ your: 'data' })
})
```

That's it. Add as many of these as you need — one per thing your frontend needs to fetch.

---

## What to do next

Swap the hardcoded `{ message: 'Hello' }` for a real ethers.js call that reads data from the blockchain:

```js
const { JsonRpcProvider, formatEther } = require('ethers')
const provider = new JsonRpcProvider(process.env.ALCHEMY_URL)

app.get('/api/balance/:address', async (req, res) => {
  const balance = await provider.getBalance(req.params.address)
  res.json({ eth: formatEther(balance) })
})
```

From here, every new concept — `.env` files, CORS, webhooks — is just one small addition on top of this foundation.
